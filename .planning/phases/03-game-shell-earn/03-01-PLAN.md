<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → api.games.endSession | Client supplies gameSessionId and score; userId is never accepted — derived server-side |
| postMessage origin → GameShell | Validated by NEXT_PUBLIC_ALLOWED_GAME_ORIGINS allowlist in frontend (Plan 03); Convex layer trusts caller identity only |
| endSession → awardSessionCoins | Internal boundary — awardSessionCoins is internalMutation, not reachable from external client |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | games.endSession — score arg | mitigate | Score is an event input only; coin calculation is server-side in awardSessionCoins. Client cannot influence coin amount by changing score arg beyond score→coin formula. |
| T-03-02 | Tampering | games.endSession — gameSessionId | mitigate | Ownership check: `session.userId !== appUser._id` → return null. Session cannot be used by a different user. |
| T-03-03 | Tampering | awardSessionCoins — double award | mitigate | Idempotency guard in endSession: if `session.endedAt` already set, return `session.coinsAwarded ?? 0` without re-calling awardSessionCoins. |
| T-03-04 | Spoofing | awardSessionCoins — userId arg | mitigate | userId in awardSessionCoins args comes from endSession after server-side betterAuthComponent.getAuthUser resolution. Client cannot supply userId to endSession; it is never an accepted arg. |
| T-03-05 | Elevation of Privilege | awardSessionCoins direct call | mitigate | awardSessionCoins is internalMutation — not in public api.* surface. Convex enforces this; external clients receive a 404/unknown-function error. |
| T-03-06 | Denial of Service | requireEnv throws on missing env var | accept | If env vars are unset, awardSessionCoins throws and endSession fails. This is a misconfiguration failure, not a security threat. Setup task documents the required pnpm convex env set commands. |
| T-03-07 | Information Disclosure | gameCatalog.list — public query, no auth | accept | gameCatalog is public catalog data (game names, genres, iframeUrls). No PII. Exposure is intentional — home page reads it without authentication. |
</threat_model>
