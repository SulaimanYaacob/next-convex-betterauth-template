    Exact contents:

    ```typescript
    import { NextRequest, NextResponse } from "next/server";

    // Routes that REQUIRE an authenticated session (registered or anonymous).
    // Anonymous sessions count as authenticated for the purpose of protection,
    // BUT we keep /dashboard and /settings limited to non-guest sessions in Phase 1
    // by treating them as protected; future phases may relax this for guests.
    const protectedRoutes = ["/dashboard", "/settings"];

    // Routes that authenticated users should be redirected away from.
    // Guests count as authenticated — they get redirected to / from auth pages.
    const authRoutes = ["/sign-in", "/sign-up"];

    type SessionResponse = {
      data?: { session?: unknown; user?: unknown } | null;
    };

    async function fetchSession(request: NextRequest): Promise<boolean> {
      try {
        const cookie = request.headers.get("cookie") ?? "";
        if (!cookie) return false;

        const url = new URL("/api/auth/get-session", request.url);
        const res = await fetch(url, {
          method: "GET",
          headers: { cookie },
          // Edge runtime fetch — no caching, request-scoped
          cache: "no-store",
        });
        if (!res.ok) return false;
        const json = (await res.json()) as SessionResponse;
        return !!json?.data?.session;
      } catch {
        return false;
      }
    }

    export default async function proxy(request: NextRequest) {
      const { pathname } = request.nextUrl;

      const isProtected = protectedRoutes.some((r) => pathname.startsWith(r));
      const isAuthRoute = authRoutes.some((r) => pathname.startsWith(r));

      // Skip the HTTP session check for routes that don't need it
      if (!isProtected && !isAuthRoute) {
        return NextResponse.next();
      }

      const isAuthenticated = await fetchSession(request);

      // Authenticated users (including guests) bounce away from sign-in / sign-up
      if (isAuthenticated && isAuthRoute) {
        return NextResponse.redirect(new URL("/", request.url));
      }

      // Unauthenticated users cannot access /dashboard or /settings
      if (!isAuthenticated && isProtected) {
        return NextResponse.redirect(new URL("/sign-in", request.url));
      }

      return NextResponse.next();
    }

    export const config = {
      // Run middleware on all routes except static assets and api routes
      matcher: ["/((?!.*\\..*|_next|api/auth).*)", "/", "/trpc(.*)"],
    };
    ```

    Notes:
    - The matcher block is BYTE-FOR-BYTE the same as the current stub — do not modify it.
    - The early-return `if (!isProtected && !isAuthRoute)` avoids paying for an HTTP roundtrip on every request to `/`, static assets that slipped through, or any other unprotected route. Only `/sign-in`, `/sign-up`, `/dashboard`, `/settings` make the call.
    - `fetchSession` returns `false` on any error (network blip, missing cookie, malformed response) — fail-closed for protected routes (sends to /sign-in) and fail-open for auth routes (lets the user see the sign-in page if uncertain). Both are the correct UX for ambiguity.
    - `cache: "no-store"` is required so each request validates fresh; otherwise edge cache could serve stale auth state.
    - Anonymous sessions return `data.session = {...}` truthy, so `isAuthenticated === true` for guests. Guests ARE bounced from `/sign-in`/`/sign-up` (D-02: after guest sign-in, user lands on `/`). They are also blocked from `/dashboard`/`/settings` — Phase 1 dashboards are auth-only scaffolds.
    - No `console.log` statements — those were debug-only in the stub.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser cookie → Edge proxy | Session cookie crosses from client to Next.js edge runtime; proxy must validate via server, not parse locally |
| Edge proxy → /api/auth/get-session | Internal HTTP call carrying forwarded cookie; trusts the response shape from Better Auth |
| Static-asset paths → matcher | Matcher must continue to exclude _next assets and api/auth routes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-19 | Elevation of Privilege | Unauthenticated access to /dashboard or /settings | mitigate | proxy.ts now actively redirects to /sign-in when `fetchSession` returns false. Replaces the stub that called getSessionCookie but never redirected (RESEARCH Pitfall 2). Verified by grep of NextResponse.redirect plus protectedRoutes array. |
| T-01-20 | Tampering | Forged cookie bypasses local-only parse | mitigate | Session validated via HTTP call to /api/auth/get-session — Better Auth verifies JWT signature server-side. Local cookie parsing rejected per CLAUDE.md guidance. |
| T-01-21 | Information Disclosure | Static assets accidentally proxied (auth check on /favicon.ico) | mitigate | Matcher pattern `/((?!.*\\..*|_next|api/auth).*)` excludes anything with a dot (favicon, images, fonts) and _next/api/auth — preserved byte-for-byte from current stub. |
| T-01-22 | Denial of Service | Edge HTTP call latency on every request | mitigate | Early-return in proxy skips the HTTP roundtrip for routes that are neither in `protectedRoutes` nor `authRoutes`. Only 4 path prefixes pay the cost. `cache: "no-store"` prevents stale auth state but does not amplify load. |
| T-01-23 | Tampering | fetch() returning malformed JSON crashes proxy | mitigate | `try/catch` around fetch + JSON parse; on any exception, returns `false` (fail-closed for protected routes, redirecting to /sign-in). |
